# 数据链路层几个基本概念

## 数据发送模型

![数据发送模型](Pasted%20image%2020241223210246.png)

1. **数据**（报文）在 H1 应用层发送给 H2 应用层需要经过红线
2. **数据**（报文）首先在运输层**分段**加上发送给 H1 **对应程序的端口**及**自己的端口**变成**报文段**
3. **报文段**(数据包、段)经过网络层时加上 H2 的 **ip 地址**变成**IP 数据报**
4. **IP 数据报**（分组、包）经过数据链路层加上**自己的 Mac 地址**和**目的主机的 Mac 地址**封装成**帧**
5. **帧**在物理层变成**比特流**
6. **比特流**在数据链路层变回**帧**，如 R1 会检查帧的**目的 Mac 地址**是否是自己，如果是则下一步，不是则丢弃
7. **帧**在网络层变回 **IP 数据报**，如 R1 会检查**目的 IP 地址**来选择对应的接口来转发
   ...

## 链路与数据链路

==链路==：物理线路<br>==数据链路==：物理线路+通信协议
 - 通信协议通常是通过网卡实现的
