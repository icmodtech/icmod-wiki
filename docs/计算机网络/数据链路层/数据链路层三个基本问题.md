# 数据链路层三个基本问题

## 封装成帧

![封装成帧](Pasted%20image%2020241223220427.png)
==MTU（最大传输单元）==：不能超过 1500 个字节<br>==帧定界（帧首 SOH、尾部 EOT）==：对不完整的帧选择丢弃，如只收到帧首部或者帧尾部
- 帧的总长度是要算上帧首尾部的
## 透明传输

==字节填充==：在帧的数据部分可能出现与帧定界相同的字符，这时就需要在相同字符前插入**转义字符** ESC

## 差错检测

==比特差错==：数据在传输过程中 1 可能变成 0,0 可能变成 1<br>==误码率==：传输数据的错误比特数与总比特数之比<br>==循环冗余检验 CRC==：

![冗余码计算](Pasted%20image%2020241223222652.png)

- P (除数)：两主机共同协商的一个二进制数，长度为 n+1
- 2<sup>n</sup>M (被除数)：n 是共同协商的，2<sup>n</sup>乘以一个二进制数为左移 n 为，后面补零。如，上图 n=3
- Q (商)：没用，要传输时会丢弃
- R (余数)（冗余码 FCS）：替换原来被除数补领部分
- 运算规律：异或（相同为 1，相异为 0）
- 结果：在两台主机共商完除数后，会对数据左移除数长度-1 位，然后做二进制除法，接着把余数加上被除数传输给另外一台主机
- 检验过程：另外一台主机收到数据后（已经插入了冗余码）把它除以事先商量的除数，如果余数为 0 则无
